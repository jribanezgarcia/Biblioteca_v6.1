Nuestro cliente está muy satisfecho de como está quedando la aplicación. Sin embargo, nos acaba de comentar que por favor añadamos persistencia a los datos introducidos, ya que tal y como está ahora la aplicación no es funcional. Por tanto, en este quinto spring añadiremos persistencia a los datos utilizando para ello bases de datos relacionales. Por tal motivo, sustituiremos el modelo que teníamos de memoria, por un modelo de base de datos.

En definitiva, lo que nos pide el cliente es lo siguiente:

    Que la aplicación almacene los datos en una base de datos creada para tal efecto.
    La base de datos debe estar implementada en MySQL.
    Adaptar el proyecto para que Gradle gestione la dependencia con el driver para java de MySQL en su última versión.
    Gestionar los usuarios para que su persistencia se lleve a cabo por medio de dicha BD.
    Gestionar los libros/audiolibros para que su persistencia se lleve a cabo por medio de dicha BD.
    Gestionar los préstamos para que su persistencia se lleve a cabo por medio de dicha BD.

Para todo esto, te iré explicando los diferentes pasos a realizar:

Primeros Pasos

    Crea un proyecto Gradle en IntelIJ y modifica el fichero de configuración para que incluya entre sus dependencias la versión del driver de mysql que se usará en el proyecto o una superior a la indicada:

    // Driver MySQL
    implementation 'mysql:mysql-connector-java:8.0.28'

Creación de base de datos MySQL en local

    Instala y ejecuta un servidor MySQL en tu equipo (por ejemplo mediante XAMPP, WAMP o instalación directa de MySQL Server).
    Accede a MySQL mediante:
        MySQL Workbench
        phpMyAdmin
        o línea de comandos
    Crea una nueva base de datos llamada: dbbiblioteca
    Utiliza el script SQL (sql - 4340 B) proporcionado (audiolibro en tabla separada) para crear las siguientes tablas: usuario, direccion, autor, libro, audiolibro, libro_autor, prestamo
    Crea un nuevo usuario en MySQL:

    Usuario: admin
    Password: biblioteca-2026

Este usuario deberá tener permisos de lectura y escritura sobre la base de datos "dbbiblioteca".

NOTA. Si trabajas en entorno local, el HOST será normalmente "localhost".

3.2 Clases de negocio para la persistencia con MySQL

    Modifica las clases de negocio para que en sus métodos se realizan las consultas necesarias a la base de datos.
    Cada clase deberá:
        Abrir conexión en comenzar()
        Cerrar conexión en terminar()
        Implementar el patrón Singleton para que sus métodos sea accesibles desde otras clase de negocio.
    Gestión específica de cada clase
        Usuarios. Debe gestionar: Tabla usuario y Tabla direccion.
        Libros. Debe gestionar: Tabla libro, Tabla audiolibro, Tabla libro_autor, Tabla autor.
        Prestamos. Debe gestionar: Tabla prestamo.
